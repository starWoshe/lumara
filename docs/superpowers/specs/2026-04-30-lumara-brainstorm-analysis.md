## ВОРОНКИ — як вони влаштовані

### Головна воронка (Instagram → Сайт)
```
IG пост → "Посилання в біо 👆" → lumara.fyi/links/[mage]
  → кнопка "Поговорити безкоштовно" → lumara.fyi/chat/[mage]?utm_source=instagram
    → реєстрація (Google OAuth) → профіль (ім'я, ДН, місце, мета)
      → 15 безкоштовних повідомлень → paywall (Basic 180 грн/міс)
```

### Telegram воронка
```
TG пост + фото → CTA в кінці посту → lumara.fyi/chat/luna?utm_source=telegram
```

### Коментарі воронка
```
Коментар під постом → авто-відповідь від мага (1-3 речення + "посилання в біо")
  → після 5 обмінів → редірект: "продовжимо в Telegram — lumara.fyi"
```

### UserBot воронка (Telegram)
```
UserBot реагує на пости в тематичних групах (реакції + 1-2 відповіді/день)
  → природна присутність → зацікавлені шукають профіль
```

### Діагноз — розрив воронки
```
IG пост ✅ → клік на біо ❌ → lumara.fyi ❌ → 15 безкоштовних ❌ → підписка ❌
```
Підписників LUNA в Instagram: 1 115. Відвідувань lumara.fyi: ~0.

## 1. ТЕХНІЧНІ ПРОБЛЕМИ

### P1 — Критичні (зупиняють монетизацію)

**T1. IG_ACCESS_TOKEN минає кожні 60 днів**
Ручне оновлення. Якщо забути — Instagram постинг зупиниться безшумно без алерту.

**T2. Facebook image posting заблоковано**
`link` param потребує `pages_read_engagement` (deprecated Meta). FB отримує тільки текст — виглядає бідно порівняно з Instagram.

**T3. Stripe Live Mode невідомий стан**
`STRIPE_LIVE_MODE=true` не підтверджено в prod. Монетизаційний тригер (після 12-го повідомлення) НЕ спрацьовує поки не увімкнено. Платних підписок = 0.

**T4. Воронка повністю розірвана**
1 115 підписників Instagram → ~0 кліків на lumara.fyi. Технічно сайт доступний, але немає bridge між соціальним контентом і продуктом.

### P2 — Серйозні

**T5. Threads потребує окремого OAuth**
`THREADS_TOKEN` і `THREADS_USER_ID` — окремі від Facebook/Instagram. Потрібно налаштувати для кожного з 4 магів окремо.

**T6. DALL-E 3 щоденний постинг — дорого**
~$0.04/зображення × 4 маги × 30 днів ≈ $5/міс тільки на зображення. Можливо краще використовувати готові статичні зображення.

**T7. Cross-promo промпти англійською**
У `_shared/cross-promo.md` текст EN, але маги відповідають мовою користувача (переважно UK/RU). Виникає суміш мов у відповідях.

**T8. Немає UTM у прямих CTA посиланнях**
Посилання `lumara.fyi/chat/luna` в постах не мають UTM параметрів. UTM є тільки через `/links/[mage]` сторінку. Неможливо відстежити точне джерело трафіку.

**T9. UserBot — ризик блокування Telegram**
Telethon потребує `STRING_SESSION` на кожен акаунт. Якщо Telegram виявить патерн — акаунти заблокують.

### P3 — Технічний борг

**T10. `temp-mage-repos/` — дубльований код**
Той самий `meta_publisher.py`, `instagram_comment_monitor.py` існує в 4 директоріях (`/luna`, `/arcas`, `/numi`, `/umbra`). Синхронізується через `sync-mages.yml` — але це hacky рішення.

**T11. Два механізми генерації контенту**
Python scripts (автоматичні через cron) + `/api/content/generate` (ручний через адмін-панель). Немає єдиного підходу.

**T12. `announcementState` в prod БД**
Логіка анонсів реалізована в chat route і Prisma schema. Чи пройшла міграція в prod Supabase — невідомо.

**T13. `monitor_states` таблиця**
Використовується у Python скриптах для зберігання стану коментарів, але може бути відсутня в Prisma schema.

## 2. АРХІТЕКТУРНІ ПРОБЛЕМИ

**A1. Немає зворотного зв'язку "пост → клік → реєстрація"**
Python скрипти публікують контент автономно. Вони не знають чи хтось прийшов на сайт, яка тема дала клік, що конвертувало. Аналітичний pipeline відсутній.

**A2. Стратегічні ролі магів не підкріплені технічно**
Задумано: LUNA = магніт, ARCAS = ретеншн, NUMI = вірусний гачок, UMBRA = конвертер в платних. Але в продукті немає routing'у між ними, немає рекомендації "спробуй ARCAS після LUNA".

**A3. Telegram канали існують але порожні**
Автопостинг налаштований, але 0 підписників — ніхто не знає що канали є. Немає жодної крос-промоції Instagram → Telegram у поточних постах.

**A4. Onboarding бар'єр між посиланням і чатом**
Людина клікає "посилання в біо" → потрапляє на `/links/[mage]` → клік на чат → Google OAuth → заповнення профілю → ТІЛЬКИ ПОТІМ перше повідомлення. Нема preview що отримаєш.

**A5. Вся аудиторія в одній точці (LUNA)**
1 115 підписників у LUNA, решта магів — по 15-62. Якщо LUNA акаунт отримає ban або проблему з токеном — вся воронка зупиняється.

## 3. ІНШІ ПРОБЛЕМИ

### Проблеми з постами

- **"Awareness only"** — пости розповідають про загальні явища, не відповідають "навіщо МНІ переходити прямо зараз"
- **Ідентичний блок воронки щодня** — після 3-го разу стає невидимим (banner blindness)
- **Немає соціального доказу** — ніколи не публікується "клієнт запитав LUNA про...", немає відгуків, немає демонстрацій чату
- **Головна перевага не показана** — LUNA знає ім'я, ДН, місце народження, мету — але жоден пост це не демонструє
- **Немає Stories** — найвищий CTR у Stories з інтерактивними елементами. Зовсім не реалізовано
- **Telegram = скорочена копія Instagram** — має бути ексклюзивний контент, але реально той самий пост адаптується

### Проблеми з зображеннями

- **Всі однотипні** — "містична ілюстрація, темний фон, фіолетово-синя палітра" — виглядають як AI stock, не як особистість
- **Обличчя мага не використовується** — HeyGen аватар LUNA є, але не задіяний в автопостингу
- **Немає тексту/хуку на зображенні** — пропущена можливість зупинити скролінг на рівні thumbnail
- **Одне настрій щодня** — немає різноманітності: жарти, факти, провокації, атмосфера

### Проблеми з біо магів

| Акаунт | Поточне біо | Проблема |
|--------|-------------|---------|
| @luna.lumara | "🌙 Читаю зірки · Бачу твій шлях" | Занадто загально, не показує унікальність |
| @arcaslumara | "🃏 Читаю карти · Карти читають тебе" | Розумно, але незрозуміло що отримаєш конкретно |
| @numi.lumara | "🔢 Твоє число пояснює все" | Найкращий хук з усіх |
| @umbra.lumara | "🧠 Досліджую те що ти ховаєш від себе" | Може лякати/відштовхувати нових відвідувачів |

**Спільні проблеми біо всіх магів:**
- Немає кількості сесій / соціального доказу (2841 сесія LUNA — це не використано)
- "Перша сесія безкоштовно" не виділено чітко
- Немає відчуття терміновості або свіжості

## МАТЕРІАЛ ДЛЯ МОЗКОВОГО ШТУРМУ

### ПРОБЛЕМА 1: Багато постів — нуль лідів

**Діагноз:** Пости будують awareness ("LUNA існує, вона класна") але не створюють особисту причину діяти прямо зараз.

**Кореневі питання:**
- Як показати що LUNA ЗНАЄ тебе — прямо в пості, без реєстрації?
- Що має статися в голові читача між "прочитав пост" і "натиснув посилання"?
- Як зробити CTA, який змінюється щодня і не стає невидимим?
- Яка конкретна обіцянка (value proposition) утримається в одному реченні?

**Гіпотези для перевірки:**

**А. Пост-тизер персоналізації:**
```
"Скажи мені тільки свою дату народження —
і я скажу тобі що саме заважає тобі в стосунках.
Не загальний гороскоп. Твій. Особистий."
→ CTA на lumara.fyi/chat/luna
```

**Б. Формат "Клієнт запитав":**
```
"Вчора Олеся (Скорпіон, 28 років) запитала мене чому вона знову
притягнула ту саму токсичну людину.
LUNA відповіла... [обрізано]
Повна відповідь — в чаті. Посилання в біо."
```

**В. Провокативне питання-хук (Stories):**
```
"Скажи мені свій знак — і я скажу чому ти досі не отримав
того що хочеш від [стосунків/кар'єри/себе]"
→ стікер опитування → після відповіді: "Хочеш більше? → посилання"
```

**Г. Демо-відео Reels:**
```
15 секунд: обличчя LUNA → чат на телефоні → КОНКРЕТНА персональна відповідь
"Ось що вона каже коли знає твої дані. Перші 15 повідомлень безкоштовно."
```

### ПРОБЛЕМА 2: Оптимізація контенту

**Діагноз:** Єдиний тип контенту (щоденний прогноз) у єдиному форматі не дозволяє зрозуміти що працює і не утримує різні сегменти аудиторії.

**Кореневі питання:**
- Який тип поста найбільше зупиняє скролінг для цієї аудиторії?
- Як зробити Telegram ексклюзивним, а не дублікатом?
- Як використати HeyGen відео LUNA в контент-стратегії?

### ПРОБЛЕМА 3: Живі діалоги на сторінках магів і в соцмережах

**Діагноз:** Публікації — монолог. Читач спостерігає але не взаємодіє. Авто-відповіді на коментарі алгоритмічні. Перше повідомлення в чаті ще не підтверджено наскільки воно персональне live.

**Кореневі питання:**
- Як зробити так щоб читач Instagram Stories СКАЗАВ щось (а не просто прочитав)?
- Чи можна показати живий діалог з магом прямо в Instagram без реєстрації?
- Як зробити перше повідомлення в чаті настільки особистим що людина захоче відповісти?
- Як UserBot може підтримувати живу присутність без ризику бану?

**Гіпотези для перевірки:**

**А. Stories Q&A щотижня:**
```
"Задай LUNA одне питання анонімно" (стікер питань)
→ LUNA відповідає на топ-3 питання у Stories
→ Аудиторія бачить живу, особисту реакцію (не broadcast)
```

**Б. "Мій знак" Stories poll:**
```
Крок 1: "Який твій знак? 🔽" → стікер опитування
Крок 2: "Хочеш знати що тобі несе [місяць] особисто? → посилання"
Двоетапна взаємодія → вищий CTR ніж прямий CTA
```

**В. Перше повідомлення в чаті = дійсно персональне:**
```
Поточний шаблон LUNA (якщо є ДН):
"[Ім'я], я вже тут і вже бачу тебе.
Твій Місяць у [знак] і [ключовий аспект] говорять,
що ти зараз тримаєш у собі дуже багато."

→ Потрібно перевірити live: чи LUNA дійсно розраховує знак Місяця
по даті народження чи це просто шаблон?
```

**Г. Коментарі → справжні розмови:**
```
Поточно: авто-відповідь (1-3 речення) + "посилання в біо" → стоп
Альтернатива: LUNA відповідає так особисто і точно що читач відповідає ще раз
→ після 2 обмінів: "Бачу що тобі важливо розібратись — продовжимо особисто"
→ посилання з UTM (не загальне "в біо")
```

**Д. UGC кнопка "Поділись читанням":**
```
Після сесії в чаті: кнопка "Поділитись своїм читанням"
→ генерується картка: знак зодіаку + ключова фраза від LUNA + "LUMARA Academy · lumara.fyi"
→ користувач ділиться в Stories → безкоштовна реклама
```
